Unity Inspector & Scripting Attributes Cheat Sheet

A comprehensive list of built-in and common custom attributes you can use in Unity.

- Field-Level Attributes

Attribute Description Requirements
[SerializeField] Serialize a private field so it appears in the Inspector. Unity core
[HideInInspector] Hide a public or serialized field from the Inspector. Unity core
[NonSerialized] Prevent a public field from being serialized by Unity. Unity core
[FormerlySerializedAs("oldName")] Remap serialized data from an old field name after renaming. Unity 2017.3+
[Header("Title")] Draw a bold header label above this field or group. Unity core
[Space] / [Space(n)] Insert vertical space (default 8ppx) before the next field. Unity core
[Tooltip("…")] Show hover-popup help text for the field label in the Inspector. Unity core
[Range(min, max)] Render a slider to clamp numeric values between min and max. Unity core
[Min(value)] Clamp numeric values to a minimum (no slider). Unity 2019.3+
[Max(value)] Clamp numeric values to a maximum (no slider). Unity 2020.1+
[TextArea(min, max)] Render a multi-line text box between min and max lines. Unity core
[Multiline] Render a simple multi-line text area. Unity core
[ColorUsage(showAlpha, hdr)] For Color, toggle alpha slider and HDR picker. Unity 2017.1+
[Delayed] Delay committing changes to string/numeric fields until focus is lost. Unity 2019.3+
[SerializeReference] Serialize polymorphic references (abstract/interfaces). Unity 2019.3+
[InspectorName("Name")] Override the label shown in the Inspector. Unity 2018.1+
[ContextMenuItem("Title","Method")] Add a small button next to the field that calls Method. Unity core
[NonReorderable] Prevent list reordering UI on a List. Unity 2020.1+

- Component & Class-Level Attributes

Attribute Description Requirements
[AddComponentMenu("…")] Place this component under a custom submenu. Unity core
[DisallowMultipleComponent] Prevent duplicates of this component on one GameObject. Unity core
[RequireComponent(typeof(X))] Auto-add or enforce component X on the same GameObject. Unity core
[ExecuteInEditMode] Run this MonoBehaviour in Edit mode. Unity core
[ExecuteAlways] Execute in Editor even when disabled or in prefab mode. Unity 2018.1+
[SelectionBase] Scene-view clicks on children select this GameObject. Unity core
[HelpURL("url")] Add a “?” icon linking to docs. Unity core
[DefaultExecutionOrder(n)] Control script execution order. Unity 2017.2+
[Serializable] Allow a class/struct to be serialized by Unity. Unity core
[AssemblyIsEditorAssembly] Exclude this assembly from player builds. Unity core
[MovedFrom(...)] API updater remaps types/namespaces after moving. Unity 2019.3+

- Method-Level & Editor Attributes

Attribute Description Requirements
[ContextMenu("Title")] Add a right-click menu to invoke this method in the Inspector. Unity core
[MenuItem("MyMenu/Action")] Add a top-bar menu item for this static method. Unity core
[InitializeOnLoadMethod] Editor: run this static method when the Editor loads. Unity 2017.1+
[RuntimeInitializeOnLoadMethod] Run this static method before any scene loads in playmode/builds. Unity core

- DOTS & Native / Burst Attributes

Attribute Description Requirements
[BurstCompile] Compile a job/function with Burst for high performance. com.unity.burst 1.0+, Jobs
[BurstDiscard] Exclude a method from Burst compilation. com.unity.burst
[WriteOnly] DOTS: mark a NativeArray as write-only. com.unity.collections, Jobs
[ReadOnly] DOTS: mark a NativeArray as read-only. com.unity.collections, Jobs
[DeallocateOnJobCompletion] Free a native collection when a job completes. com.unity.collections, Jobs
[NativeDisableParallelForRestriction] Allow unsafe parallel writes in jobs. com.unity.collections, Jobs
[NativeDisableUnsafePtrRestriction] Skip safety checks on pointers in jobs. com.unity.collections, Jobs
[RequiredByNativeCode] Mark code used by Unity’s native engine (avoid stripping). UnityEngine.Scripting
[Preserve] Prevent IL2CPP or managed stripping on this code. UnityEngine.Scripting

- C# Language & Interop Attributes

Attribute Description Requirements
[Serializable] Allow .NET serialization (used by JsonUtility). .NET
[Obsolete("msg")] Mark code as deprecated with a warning or error. .NET
[Conditional("UNITY_EDITOR")] Compile method calls only when symbol is defined. .NET
[DllImport("…")] Declare external native functions (P/Invoke). .NET

- Custom Property Drawers

?

Define your own PropertyAttribute + PropertyDrawer to create custom Inspector widgets: 


        // 1) Create the attribute
        public class MyRange : PropertyAttribute
        {
            public float min, max;
            public MyRange(float min, float max)
            {
                this.min = min;
                this.max = max;
            }
        }
        
        // 2) Create the drawer
        #if UNITY_EDITOR
        [CustomPropertyDrawer(typeof(MyRange))]
        public class MyRangeDrawer : PropertyDrawer
        {
            public override void OnGUI(Rect position, SerializedProperty property, GUIContent label)
            {
                var range = (MyRange)attribute;
                property.floatValue = EditorGUI.Slider(
                    position,
                    label,
                    property.floatValue,
                    range.min,
                    range.max
                );
            }
        }
        #endif
        
        // 3) Use it
        public class Example : MonoBehaviour
        {
            [MyRange(0f, 1f)]
            public float blend;
        }

- Editor Formatting Control

Prevent automatic code formatting in Visual Studio or Rider (with ReSharper) for specific blocks of code:

- ReSharper Formatter Directives

?

These comments tell ReSharper (used in JetBrains Rider or as a Visual Studio plugin) to skip formatting between the markers:

// @formatter:off
var  weirdlyFormatted   =   "leave   me    alone!";
// @formatter:on

Note: To use this in Visual Studio, ReSharper must be installed and “Enable formatter markers” must be enabled in ReSharper → Options → Code Editing → C# → Formatting Style → General.

Does not work in CSS.

- Visual Studio

?

Visual Studio’s native formatter does not support formatter-off comments. Instead, you can mark areas with #region to locate them easily:

#region DoNotFormat
var  tempHack  =   "manually     preserved";
// Add comments or notes here
#endregion
